# Caching Blog API

REST API для блога с реализацией CRUD операций и кешированием популярных постов. Проект построен на принципах чистой архитектуры, сопровождается логгированием и ограничением количества запросов.

## Подход к кэшированию

Кэширование обеспечивается за счёт Redis. При запросе публикации система ищет данные в кэше, если они не найдены, переходит к БД. После запроса данные сохраняются в кэше, чтобы лишний запрос к БД не поступал. Время хранения данных 10 минут. При обновлении и удалении публикации происходит инвалидация данных из кэша, если они есть.

## Переменные окружения

Проект предоставляет сразу 2 файла окружения - для основного приложения и для тестов.

**Структура переменных окружения основного приложения**

* **POSTGRES_USER**
* **POSTGRES_PASSWORD**
* **POSTGRES_DB**
* **DATABASE_URL**
* **REDIS_URL**

**Структура переменных окружения для тестов**

* **DATABASE_URL**
* **REDIS_URL**

## Быстрый старт (запуск с Docker)


### 1. Клонирование репозитория

Склонируйте проект с GitHub и перейдите в его директорию:

```bash
git clone https://github.com/ArtemW2/Caching-Blog.git
cd Caching-Blog
```

## 2. Запуск контейнеров

Предоставлено 2 контейнера - основное приложение и тесты

```bash
docker-compose up -d --build
docker-compose -f docker-compose.test.yaml up -d --build
```

## 3. Запуск тестов

Предоставлено 22 теста - интеграционные и юнит

```bash
pytest tests/ -v 
```
